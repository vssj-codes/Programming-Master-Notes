# Python Django Tutorial: How to Use a Custom Domain Name for Our Application

## Key Insights

- **Introduction**
  - Learn to set up a custom domain name for your Django application.
  - Use NameCheap as the domain registrar and Linode for hosting.

- **Registering a Domain with NameCheap**
  - Sign in to your NameCheap account or create one.
  - Search for your desired domain and purchase it.
  - After purchasing, navigate to the "Domain List" and select "Manage" for your domain.

- **Configuring DNS Settings**
  - In NameCheap, go to the "Advanced DNS" tab.
  - Add an "A Record" for your domain pointing to the IP address of your Linode server.
  - Example:
    ```plaintext
    Type: A Record
    Host: @
    Value: <Your Linode IP Address>
    TTL: Automatic
    ```

- **Setting Up Linode Server**
  - Log in to your Linode account and create a new Linode instance.
  - Choose an appropriate plan and set up the instance.
  - SSH into your Linode server and update the server packages:
    ```bash
    sudo apt update
    sudo apt upgrade
    ```

- **Installing and Configuring Nginx**
  - Install Nginx on your Linode server:
    ```bash
    sudo apt install nginx
    ```
  - Start and enable Nginx:
    ```bash
    sudo systemctl start nginx
    sudo systemctl enable nginx
    ```

- **Configuring Nginx for Django Application**
  - Navigate to the Nginx sites-available directory:
    ```bash
    cd /etc/nginx/sites-available/
    ```
  - Create a new configuration file for your domain:
    ```bash
    sudo nano yourdomain.com
    ```
  - Add the following configuration:
    ```nginx
    server {
        listen 80;
        server_name yourdomain.com www.yourdomain.com;

        location / {
            proxy_pass http://127.0.0.1:8000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }
    ```
  - Create a symbolic link to enable the site:
    ```bash
    sudo ln -s /etc/nginx/sites-available/yourdomain.com /etc/nginx/sites-enabled/
    ```
  - Test the Nginx configuration and restart the service:
    ```bash
    sudo nginx -t
    sudo systemctl restart nginx
    ```

- **Configuring Django Settings**
  - Update `ALLOWED_HOSTS` in your Django `settings.py`:
    ```python
    ALLOWED_HOSTS = ['yourdomain.com', 'www.yourdomain.com']
    ```
  - Ensure your application is running and accessible through the configured domain.

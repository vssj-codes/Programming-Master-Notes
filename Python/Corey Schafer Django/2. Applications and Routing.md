# Python Django Tutorial: Full-Featured Web App Part 2 - Applications and Routes - YouTube

## Key Insights

- **Creating a Blog App**
  - In Django, a project can contain multiple apps, each representing a different part of the website (e.g., blog, store).
  - Use the command `python manage.py startapp blog` to create a new blog app within your project.

- **Project Structure**
  - The `startapp` command creates a new directory (blog) with several files, including `views.py`.
  - Django’s structure might seem complex initially due to many autogenerated files, but it provides more out-of-the-box functionality compared to smaller frameworks like Flask.

- **Creating a View**
  - Open `views.py` in the blog directory and define a function to handle traffic for a specific route.
  - Example:
    ```python
    from django.http import HttpResponse

    def home(request):
        return HttpResponse('<h1>Blog Home</h1>')
    ```

- **Mapping URLs to Views**
  - Create a `urls.py` file in the blog directory to map URLs to view functions.
  - Example:
    ```python
    from django.urls import path
    from . import views

    urlpatterns = [
        path('', views.home, name='blog-home'),
    ]
    ```

- **Including Blog URLs in Project URLs**
  - Update the project's main `urls.py` to include the blog URLs.
  - Example:
    ```python
    from django.urls import path, include
    # trailing slashes /: It is a good idea to leave it, by default if it has a trailing slash django will redirect routes without / to that route that has one. So blog/ and blog (without/) will get redirected to blog routes, that's usually what we want in websites so no one ever gets confused about what routes are not returning the same results 
    urlpatterns = [
        path('admin/', admin.site.urls),
        path('blog/', include('blog.urls')),
    ]
    ```

- **Running the Server**
  - Start the Django development server using `python manage.py runserver` and navigate to the specified routes to test.

- **Creating Additional Routes**
  - Define more view functions and corresponding URL patterns for other pages (e.g., about page).
  - Example for about page:
    ```python
    # In views.py
    def about(request):
        return HttpResponse('<h1>Blog About</h1>')

    # In urls.py
    urlpatterns = [
        path('', views.home, name='blog-home'),
        path('about/', views.about, name='blog-about'),
    ]
    ```

- **Trailing Slashes in URLs**
  - Django automatically handles trailing slashes, redirecting URLs without a trailing slash to the ones with it.

- **Setting the Blog as the Homepage**
  - To make the blog the homepage of the entire site, modify the project’s `urls.py` to point the root URL to the blog app.
  - Example:
    ```python
    urlpatterns = [
        path('', include('blog.urls')),
    ]
    ```

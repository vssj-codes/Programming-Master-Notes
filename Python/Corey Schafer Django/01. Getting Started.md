# Python Django Tutorial: Full-Featured Web App Part 1 - Getting Started

#### Key Insights:
- **Introduction**:
  - Building a full-featured web application using Django.
  - The application will be a blog-style app with user authentication, profile updates, and CRUD operations for posts.

- **Project Setup**:
  - Create a new Django project using `django-admin startproject projectname`.
  - Project structure includes:
    - `manage.py`: Command-line utility for Django projects.
    - `projectname/` directory:
      - `__init__.py`: Marks the directory as a Python package.
      - `settings.py`: Configuration settings for the project.
      - `urls.py`: URL routing for the project.
      - `wsgi.py`: Web Server Gateway Interface configuration.

- **Installing Django**:
  - Use `pip install django` to install Django.
  - Verify the installation with `python -m django --version`.

- **Running the Server**:
  - Use `python manage.py runserver` to start the development server.
  - Access the default site at `http://127.0.0.1:8000` or `http://localhost:8000`.

- **URL Routing**:
  - The default `urls.py` includes a route for the admin interface.
  - Example:
    ```python
    from django.contrib import admin
    from django.urls import path

    urlpatterns = [
        path('admin/', admin.site.urls),
    ]
    ```

- **Admin Interface**:
  - Accessible at `/admin` once the server is running.
  - Requires superuser credentials to access.

- **Project Configuration**:
  - `settings.py` contains essential configurations like `SECRET_KEY`, `DEBUG`, `INSTALLED_APPS`, `DATABASES`.
  - Modify these settings as needed for the project.
